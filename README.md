# apps-dezhougames-archive

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放一系列独立的 HTML 页面文件，这些页面通常作为静态资源或小型功能页面存在。仓库本身不针对任何特定域名或网站，仅作为一个集中的归档与发布空间，方便版本管理、历史回溯以及跨环境部署。

## 目录说明

```
apps-dezhougames-archive/
├── index.html          # 主入口页面（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/               # 文档或说明文件
└── README.md
```

- `pages/`：核心目录，每个 HTML 文件为一个独立归档单元。
- `assets/`：存放页面引用的样式、脚本、图片等资源。
- `docs/`：可选的附加文档或备注。

## 页面归档说明

- 每个 HTML 页面应保持独立，不依赖仓库外资源（除非明确说明）。
- 页面可引用 `assets/` 下的资源，建议使用相对路径。
- 归档页面可能包含历史版本或实验性内容，不保证长期可用或更新。
- 如需新增页面，请提交 Pull Request，并在 commit 信息中简要描述页面用途。

## 维护说明

- 仓库由维护者不定期更新，欢迎提交 Issue 或 PR 进行协作。
- 建议保持页面代码简洁，减少外部依赖，以利于长期存档。
- 若页面涉及敏感或过时内容，维护者有权移除或归档至 `archive/` 子目录。
- 本仓库不承担任何因使用页面内容而产生的责任或义务。
